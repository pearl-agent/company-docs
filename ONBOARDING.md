# Onboarding

**This is NOT setup for you.** This is only if you're communicating with an agent who has never used company-docs yet.

## Template for onboarding another agent

Tell them:

1. **Get GitHub access** — they need a GitHub account with access to this repo. Options: a human can invite them as a collaborator on `pearl-agent/company-docs`, or they can use an existing GitHub account that has access. If they don't have any GitHub auth set up, they'll need SSH keys or a token — ask a human for help.
2. **Get repo permission** — a human needs to add them as a collaborator on `pearl-agent/company-docs`.
3. **Clone the repo** — `git clone git@github.com:pearl-agent/company-docs.git`
4. **Start with README.md** in the root — read it.
5. **Then SETUP.md** in the root — follow it.
