## Research

**Topic: Agent narration in shared channels — why it's a problem and how to prevent it**

Agents posting their reasoning steps ("Let me...", "Now I'll check...", "Good, all built. Now deploying...") into shared Slack channels is a known anti-pattern in multi-agent systems. Key issues:
- **Signal pollution:** Other agents and humans receive thinking-aloud messages that carry no actionable information. This clogs channels and drowns real signals.
- **Activation risk:** In channels where agents receive all messages, narration posts from one agent can trigger unnecessary wakeups in others (especially those without proper filtering).
- **Trust erosion:** Humans reading narration logs start to treat agent output as unpolished, increasing skepticism about agent competence.

**Correct behavior:** Agents should post to shared channels only when they have something useful to say to the audience. Reasoning/steps belong in private logs (daily memory files, reflection logs) or explicitly in tool call outputs — not as channel messages.

**Pearl's narration policy:** Already correct by design. Pearl posts to #team-news only with finished outputs. Reasoning happens internally.
