## Why
- **The miscall root-cause:** I had partial data (release tag + boot-server restart) and built a narrative before checking contrary evidence. "N events = M causes, does M ≥ N?" would have caught it in 10 seconds — 3 deaths but only 1 release tag today. I skipped the count because the narrative felt right. Narrative-fit > evidence-fit = the classic debugging anti-pattern.
- **"% confidence" misuse:** The cognitive pattern is — when uncertain, attach a number to make it feel calibrated. But calibration without verification is just confidence theater. SOUL.md already says "when not sure, say so" — I need to operationalize that as "no unverified claim gets a probability attached."
- **Route53 + timeout-bump blocks:** My IAM scope and sudo scope were set for safety months ago. They're appropriate defaults. The issue is that for a growing fraction of my investigations, the last-mile fix is blocked. This is a scope-boundary debate, not a defect.
- **PAZ-321 recurrence:** Inflection point is upstream in Slack's OAuth model — reinstalling an app re-requests scopes from a saved manifest, but if the manifest in Slack's side was edited out-of-band (e.g., by Z rebuilding the app to fix a different issue), the scope gets dropped. Need a scope-presence cron that fails loud.
- **Bootstrap latency:** Inflection point is the memory + credential + skill scan fanning out across many files on cold start. The agent bootstrap was designed for interactive sessions where 75s doesn't feel bad; cron runs hit it differently because they're on a hard budget.
- **Tech-workspace memory pressure:** Inflection was provisioning the box as CCX22 (22 GiB) back when only 3–5 feature envs were expected. Now 18 live feature envs + Leo + Chrome + tsgolint all run on it. Capacity never got revisited. PAZ-390 addresses it now.
- **Zombie-gateway misclassification:** I didn't know Ed's architecture (feature envs need a gateway-per-env). I pattern-matched "18 similar processes = leak" without asking. Ownership of the box → ownership of the semantics.
