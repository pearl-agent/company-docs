## Notes

- `~/.codex/auth.json` schema: `{ auth_mode: "apikey"|"chatgpt", openai_api_key?: string, tokens?: { access_token, refresh_token, id_token, account_id } }`. CLI reads this on every invocation — there's no in-memory caching past process boundaries.
- Per-agent credential stores: `/home/pazi/.openclaw/agents/<agent-id>/agent/auth-profiles.json`. Sub-agent `pazi-developer`, `pazi-qa`, `pazi-tech-lead` each have one. Main agent's store is read by `loadAuthProfileStoreForSecretsRuntime()` which is what `handlePushCredentials` currently uses. Sub-agents miss out entirely.
