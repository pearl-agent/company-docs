## Research
- Query: `slack socket mode recv-q buffered events not consumed` → Slack blog + node-slack-sdk issues. Multiple reports of Socket Mode WSS clients going silent when the app blocks the read loop. Typical cause: event handler throwing synchronously inside the ACK path, or the WebSocket frame parser getting stuck on a large payload.
- Query: `slack multi-app workspace message.channels event routing` → Slack API docs. Slack explicitly fans out `message.channels` to all subscribed apps. So my "Slack only delivers to one app" theory was wrong. Each app gets its own copy. If Parker isn't processing, it's *not* because Chris "stole" the event.
- Query: `git clone exit code 128 tag not found release CI` → standard. `exit 128 → fatal: remote error: upload-pack: not our ref`. Signals the exact thing I found: the tag referenced doesn't exist upstream. Good corroboration.
- Query: `sentry project rate limit spike protection quota overage` → Sentry docs. Project-level rate limits available, configurable per issue via "inbound filters." Confirms yesterday's note.
- Query: `production alert deduplication state change vs state presence` → standard alerting lit (Google SRE book, PagerDuty docs). Cron-based alerts should fire on deltas, not absolute counts. No new insight but reinforces yesterday's idea.
