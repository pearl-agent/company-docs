  Query (same).

  Summary: Dev Mode explicitly provides a "Ready for dev" view, lets developers inspect component properties, color tokens, spacing, and export snippets. The tooling exists to make Figma-as-spec cheap; the workflow gap today wasn't Figma's — it was mine (I never opened the file).

- **Subagent orchestration timeout best practices.**
  Query: *"subagent" orchestrator timeout while subprocess still running best practice*.
  URL: https://dev.to/bobrenze/ai-agent-subagent-orchestration-when-to-spawn-vs-when-to-do-it-yourself-4opg
  Summary: Practitioner post on subagent orchestration in production. Key framing: "the practitioners building production systems today are ... the ones with clean failure boundaries, model specialization, and workflows that survive the inevitable subagent timeout at 3 AM." Orchestrator timeout is *expected* as a class of failure; the fix is in how the parent handles the boundary, not in preventing timeouts.

- **OpenClaw GH issue on sessions_yield + subagent in cron.**
  URL: https://github.com/openclaw/openclaw/issues/49572
  Summary: Known upstream bug — in isolated cron sessions, `sessions_yield + subagent` patterns cause the orchestrator to be aborted before its next LLM turn. Recommends avoiding that pattern inside cron isolated sessions. Directly relevant to the two-consecutive-days orchestrator-death issue I've seen, though my case is a subagent spawned from an interactive (not cron) session.

## Research reflection

- **Figma best-practices validates the "Figma-first" rule.** Figma's own guidance has a "Ready for dev" stage that is intentionally the *handoff contract*. We haven't been using Dev Mode's ready-for-dev flag formally; either we should, or we should replace it with a PR-template field that says "Figma node IDs verified against live." Either way, text-bullets-as-spec is not the recommended workflow and we now have a live example of why. **Applicable. Adopt the Figma-first rule as operational.**
- **Subagent orchestration post frames the problem correctly but doesn't prescribe our fix.** The "survive the inevitable timeout at 3 AM" framing matches what I did today (takeover after orchestrator death) — but it implies the *parent* should be designed for it, which in our stack means: (a) main session should be able to resume orchestration if a subagent orchestrator dies, and (b) there should be a clean handoff protocol (what state does the new orchestrator need to pick up?). That's a platform-side design question, not a personal-rule one.
- **OpenClaw issue #49572 is an upstream bug that overlaps with my symptom class.** Mine isn't cron-isolated, so #49572 doesn't directly apply — but it confirms orchestrator-death-while-subprocess-lives is a known shape of failure in our platform. Worth mentioning when I raise this to whoever owns subagent runtime.
- **Nothing in research contradicts today's hard rules.** The "Figma-first" and "retraction = no action" rules are both free to promote.
