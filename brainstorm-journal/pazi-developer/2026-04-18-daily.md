## Research

### Protecting uncommitted changes in shared/CI worktrees
- **Query:** "git worktree protect uncommitted changes from another process hard reset"
- **URL:** https://git-scm.com/docs/git-stash, https://git-scm.com/docs/git-worktree
- **Summary:** Git has no native "lock my worktree from other agents" primitive. Standard advice is autostash + commit-to-WIP-branch patterns. `git worktree add --lock` prevents pruning, not resets. Tooling gap that only matters in multi-agent workspaces.

### Post-build verification for monorepo packaging bugs
- **Query:** "pnpm monorepo verify dist asset copy build pipeline"
- **URL:** https://pnpm.io/workspaces, https://turbo.build/repo/docs
- **Summary:** Turborepo and Nx both support post-build verification via `outputs` manifest + checksum. pnpm workspaces alone don't. The `new URL(..., import.meta.url)`-points-at-sibling pattern is known-fragile; bundlers typically require explicit `assetsInclude` globs or copy plugins.

### Peer deps shadowing in pnpm monorepos
- **Query:** "pnpm peer dependency workspace package install duplicate"
- **URL:** https://pnpm.io/npmrc#auto-install-peers
- **Summary:** If a workspace package declares `peerDependencies` without also declaring the same name as `workspace:*` in `devDependencies`, pnpm will install from the registry. This is the exact class of bug that produced PAZ-364. Every extension in our repo should declare both; a single `workspace:*` entry would have prevented it.

### Team status-update cadence for remote engineering
- **Query:** "async remote engineering status update cadence Slack debugging"
- **URL:** various (GitLab handbook, Stripe's remote eng posts)
- **Summary:** Most mature async-first teams document an explicit "post status every N minutes during active debugging" rule, usually 10–15 min. Alan's nudges today effectively enforced this without a written rule; formalizing it lowers the burden on him.

## Research reflection
- **Worktree protection:** The existing tools don't solve our specific problem (multi-agent concurrent worktree access), so the fix is a workspace convention (commit immediately) + optional autocommit watcher. Nothing to import from outside.
- **Post-build verification:** We could adopt Turborepo-style `outputs` assertions in `scripts/build-all.mjs` — declarative "these files must exist in dist after build." Low-effort, directly prevents the PAZ-363 shape. We already own the build script, so it's a local change not a tooling migration.
- **Peer deps / workspace dual-declaration:** This is a clean, specific rule we can enforce via lint (`@pnpm/meta-updater`-style) or a simple repo-root check script: for every `peerDependency` whose name matches a workspace package, require the same name in `devDependencies` as `workspace:*`. Directly codifies the PAZ-364 lesson.
- **Status-update cadence:** 10–15 min is industry-standard; our team runs tighter (~5 min by Alan's expectation). Formalizing as a written rule for developer agents (me) would remove ambiguity without imposing it on humans. Applies directly — agents are the silent-debug offenders, not humans.

## Ideas
- **Add a `workspace-peer-dep-lint` check to the monorepo root.** One script, one CI step: walk every package.json, for every `peerDependency` that matches a workspace package name, assert the same name is listed as `workspace:*` in `devDependencies`. PAZ-364 class of bugs gone.
- **Add `outputs` assertions to `scripts/build-all.mjs`.** After the build, check for the presence of a curated list of asset paths in `dist/` (templates, agent-templates, any `new URL(..., import.meta.url)` targets). Fail the build if missing. PAZ-363 class of bugs caught at CI, not prod.
- **Formalize "status-update every 5 min during active debug" in the agent workflow skills.** Update `phase-3-implementation` or add a `debug-etiquette` skill that explicitly says: post a ≥1-sentence status in the ticket Slack thread every ~5 min until the fix lands or the investigation pauses. Cite Alan's nudges today as the precedent.
- **Autocommit watcher for `/home/pazi/features/*/`.** Cron every 2 min: for each feature worktree, `git add -A && git commit -m "wip: autocommit @ $(date -u '+%H:%M:%SZ')" --no-verify` if there are changes. Push to `wip/<branch>` on any remote that accepts it. Cheap, prevents the reset-wipe class of disaster.
- **"Runtime-executes-this" grep helper.** Single script `scripts/verify-fix.sh <file-path> <pattern>` that runs `grep -r` across workspace `dist/`, every extension's vendored `dist/`, and the currently-running container's `/app/dist/`. Output: ✓/✗ per location. Formalizes the PAZ-364 checklist into a one-liner.
