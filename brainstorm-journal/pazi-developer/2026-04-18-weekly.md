## Why
- **QA false-positive pattern**: QA plans are built from ticket descriptions and technical surfaces, not user stories. The QA agent can verify "endpoint returns 200" but can't evaluate "did the user see an image." This is a methodology gap, not an agent-skill gap.
- **DM gap**: Tool path broke silently; no error surfaced, no fallback attempted; conversations with a co-founder just… stopped working. The whole delivery pipeline has zero observability for "message was supposed to go out, didn't."

- **Local env fragility**: `pazi-local-setup` grew incrementally — each new feature added a workaround, not a test. No comprehensive integration check.
- **Memory hygiene**: Worker-instance writes don't propagate to main workspace's `memory/`. The workflow never wrote the "I shipped X today" trail back home.
- **Slack manifest loss**: The debugging hypothesis space was wrong (Socket Mode vs webhook, URL verification, etc.) when the real answer was a mundane boolean. Without a manifest validator, we had to guess.
- **Pre-existing red tests**: The "green on changed area" merge bar creates a boiling-frog: nobody ever has "clean up red tests" as their ticket, so it never happens.
