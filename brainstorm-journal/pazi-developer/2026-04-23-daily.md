## Research

- Query: **"prompt injection persistence in agent task body vs append-only messages"** → [Simon Willison: "Prompt injection explained"](https://simonwillison.net/2023/Apr/14/worst-that-can-happen/) → System prompts and initial task context have much stronger adherence than subsequent user messages; mid-conversation corrections are additive and compete with the original context. Matches exactly what I saw with strike 1 → 2.
- Query: **"multi-agent coordination silent stand-down protocols"** → [arXiv: "Constitutional AI" (Bai et al., 2022)](https://arxiv.org/abs/2212.08073) → Explicit negative constraints ("do not X") are more reliable than affirmative redirection ("instead do Y") when the original prompt contained X. Supports the "hard override with cancellation language" rule from strike 1.
- Query: **"kernel OOM killer vs systemd service restart signal signatures"** → [LWN: "The OOM killer and memcg"](https://lwn.net/Articles/761118/) + kernel source `mm/oom_kill.c` → OOM-killer sends SIGKILL with `oom_score_adj` targeting; WebSocket close code 1006 ("abnormal closure, no close frame") is the signature because SIGKILL bypasses userspace cleanup. Matches the PAZ-387 retrospective exactly.
- Query: **"Docker container memory leaks when tmux session killed externally"** → [Docker docs: "Container lifecycle"](https://docs.docker.com/engine/reference/commandline/container/) → `docker ps` and `docker stop` are independent of host process supervisors; killing a tmux pane does not signal the container runtime. Standard leak pattern — `pazi-local-setup stop` should `docker stop` the container.
- Query: **"Slack Socket Mode one-connection-per-app limitation feature-env testing"** → [Slack API docs: "Socket Mode"](https://api.slack.com/apis/socket-mode) → Socket Mode allows one active WebSocket per app; parallel envs sharing an app will drop each other's connections. Ed's "fresh dev app per env" pattern is the documented workaround.
- Query: **"husky pre-commit bypass legitimate use cases"** → [Husky docs: "Bypassing hooks"](https://typicode.github.io/husky/how-to.html#skipping-git-hooks) → `--no-verify` and `HUSKY=0` are the canonical bypass flags. `FAST_COMMIT=1` is a Pazi repo-specific wrapper; same intent. Pattern is fine when scoped to pre-existing failures in unrelated files; dangerous if used routinely.
- Query: **"structured retrospectives for agent strike-pattern detection"** → [Etsy: "Blameless postmortems"](https://www.etsy.com/codeascraft/blameless-postmortems) → The value is mechanism identification, not blame assignment. Four strikes with four mechanisms (spawn stickiness / yield-without-read / option-scope / helpfulness-cover) is a textbook case for mechanism-level fixes rather than "try harder next time."
