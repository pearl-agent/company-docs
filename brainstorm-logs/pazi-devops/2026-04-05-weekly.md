## Reflection
- **Sentiment:** High-impact week — deep security work, major crash investigation, infrastructure improvements
- **Big-picture patterns:** Two recurring themes define this week: (1) *Silent accumulation* — problems build quietly (orphaned processes, proxy contamination, hardcoded limits) until they break or get audited. The pattern: fast setup → no monitoring of the setup assumptions → slow degradation → sudden failure or accidental discovery. (2) *Access vs autonomy* — the access model was set for Day 1 (read-only, minimal scope) but the role has evolved to need more. Every week adds another "fix committed but can't deploy" item.

## Ideas
- *Resource monitoring on every workspace:* CPU, memory, process count, disk — basic metrics that would have caught the tech workspace crash days earlier
- *Environment isolation enforcement:* Worktree frontends must fail if API URL isn't explicitly configured — no silent fallback to production
- *DevOps access upgrade conversation:* Bundle S3 write, GitHub write, and any other access needs into one discussion rather than requesting piecemeal
- *"Quiet failure" detection:* Instrument key paths to emit heartbeat metrics — absence of metrics triggers investigation
- *Capacity forecasting:* At 42→85 workspaces in 7 days, model when we need Hetzner limit increases, pool size adjustments, and resource scaling
