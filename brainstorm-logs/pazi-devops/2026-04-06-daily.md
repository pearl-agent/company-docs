## Reflection

## Ideas
- Switch cron jobs to `sessionTarget=isolated` to prevent re-trigger loops — each run is self-contained
- Add duplicate detection: if a cron job's output is identical to the last run within X minutes, suppress it
- OpenClaw cron delivery should have an explicit ack mechanism — fire once, confirm delivery, don't retry unless failure
