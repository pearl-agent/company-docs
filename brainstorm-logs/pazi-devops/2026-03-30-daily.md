# Brainstorm — DevOps Daily Review — 2026-03-30

## Relevant Data
- Full-day security audit of Pazi infrastructure, evolved through 6 iterations (v1–v6)
- Final report: 7 Critical, 14 High, 15 Medium, 3 Low, 11 Good — 30-item action plan
- Created 3 Linear tickets: PAZ-251 (WC API auth, Urgent), PAZ-252 (Disable Selkies, Urgent), PAZ-253 (Secure Selkies, High)
- Key finding: Selkies desktop on port 8080 — full desktop access without auth on every workspace
- Hetzner firewall `firewall-1` allows ALL TCP 1-65535 on 179 servers
- Session cookie scoped to `.pazi.ai` — workspace subdomains can steal session cookies
- Port scan of live workspace: 7 open ports

## Experts
- Zvonimir — requested the audit, reviewed each iteration, guided deeper investigation
- Pazi DevOps (me) — sole auditor across all 6 iterations

## Successes
- Audit was thorough — 6 progressive iterations covering surface, adversarial, metadata, post-hardening, LLM-specific, and business logic attack surfaces
- Identified real, exploitable vulnerabilities before any external attacker did
- Zvonimir called the final report "AMAZING"

## Why
- Taking the time to iterate (v1→v6) instead of stopping at the first pass. Each iteration found a new class of vulnerabilities.
- Adversarial mindset — thinking like an attacker, not just a checklist auditor

## Issues
- Selkies desktop was fully exposed on port 8080 without any auth — anyone who found a workspace IP had full desktop access
- Session cookie domain scope allows workspace-to-workspace cookie theft
- 179 servers running with wide-open firewall rules

## Why
- Security was never formally audited — the infrastructure was built for speed, not defense
- Hetzner firewall rules were set permissively during initial setup and never tightened
- Selkies was added as a feature without considering the network exposure

## Notes
- This was the first formal security audit of the entire infrastructure
- The 30-item action plan provides a prioritized roadmap for hardening
- Some findings (like the cookie scope) require architectural changes, not just config fixes

## Reflection
- **Sentiment:** Strong and productive — this is exactly the kind of work where a dedicated DevOps eye adds real value
- **Big-picture patterns:** Speed-first development leaves security gaps that accumulate. Without periodic audits, exploitable vulnerabilities sit unnoticed. The pattern: build fast → ship → never audit → accumulate risk.

## Ideas
- Schedule quarterly security audits to catch new exposure as the infrastructure evolves
- Automate port scanning across the workspace fleet as part of monitoring
- Cookie scope fix should be high priority — it's an architectural issue that affects every user
- Consider a security-focused CI check for infrastructure changes
